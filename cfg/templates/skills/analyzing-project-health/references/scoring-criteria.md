# 评分标准详细定义

每个维度 0-100 分，基于可量化的指标。

## 1. 依赖健康度 (Dependency)

| 分值 | 条件 |
|------|------|
| 90-100 | 无过期依赖，无 CVE，无废弃包 |
| 70-89 | ≤5 个过期（非安全相关），无高危 CVE |
| 50-69 | 有中危 CVE 或 >10 个过期依赖 |
| 30-49 | 有高危 CVE 未修复 |
| 0-29 | 多个高危 CVE + 大量废弃包 |

**量化方法**：
- 检查 `package.json`/`build.gradle`/`requirements.txt` 等依赖文件
- 运行 `npm audit`/`pip audit`/依赖检查工具
- 统计过期依赖比例

## 2. 文档完整度 (Documentation)

| 分值 | 条件 |
|------|------|
| 90-100 | README + API 文档 + ADR + 内联注释齐全 |
| 70-89 | README 完整，主要 API 有文档 |
| 50-69 | README 存在但不完整，部分 API 有文档 |
| 30-49 | 仅有基础 README |
| 0-29 | 无文档或文档严重过期 |

**量化方法**：
- 公共 API 文档覆盖率
- README 包含：项目说明、安装步骤、使用示例、架构概述
- 检查是否有 ADR（架构决策记录）

## 3. 测试覆盖率 (Testing)

| 分值 | 条件 |
|------|------|
| 90-100 | 覆盖率 >80%，含集成测试，关键路径全覆盖 |
| 70-89 | 覆盖率 60-80%，核心逻辑有测试 |
| 50-69 | 覆盖率 40-60%，有明显测试缺口 |
| 30-49 | 覆盖率 <40%，仅部分模块有测试 |
| 0-29 | 基本无测试或测试大量失败 |

**量化方法**：
- 运行覆盖率工具（`jest --coverage`/`jacoco`/`pytest --cov` 等）
- 检查关键业务路径是否有对应测试
- 统计测试失败/跳过数量

## 4. 安全态势 (Security)

| 分值 | 条件 |
|------|------|
| 90-100 | 无已知漏洞，有安全审计，敏感信息管理规范 |
| 70-89 | 无高危漏洞，基本的输入验证 |
| 50-69 | 存在中危漏洞或部分 OWASP 风险 |
| 30-49 | 有高危漏洞或敏感信息泄露风险 |
| 0-29 | 多个高危漏洞 + 认证/授权缺陷 |

**量化方法**：
- 检查硬编码密钥/密码（grep 敏感模式）
- 检查 SQL 注入/XSS/CSRF 防护
- 验证认证/授权实现
- 依赖漏洞扫描结果

## 5. 架构质量 (Architecture)

| 分值 | 条件 |
|------|------|
| 90-100 | 清晰分层，无循环依赖，模块高内聚低耦合 |
| 70-89 | 分层基本合理，≤2 个循环依赖 |
| 50-69 | 存在分层违规或 3-5 个循环依赖 |
| 30-49 | 架构混乱，多处分层违规 |
| 0-29 | 无明确架构，严重耦合 |

**量化方法**：
- 检查模块间依赖方向（是否违反分层）
- 检测循环依赖（工具或代码分析）
- 评估模块职责是否单一

## 6. 代码质量 (Code Quality)

| 分值 | 条件 |
|------|------|
| 90-100 | 圈复杂度 <10，重复率 <3%，无明显 Code Smell |
| 70-89 | 圈复杂度偶有超标，重复率 <5% |
| 50-69 | 多个函数复杂度 >15，重复率 5-10% |
| 30-49 | 大量高复杂度函数，重复率 >10% |
| 0-29 | 代码质量极差，大量坏味道 |

**量化方法**：
- 检查函数长度（目标 ≤30 行）
- 计算圈复杂度（目标 <10）
- 检测重复代码块
- 检查嵌套层级（目标 ≤3 层）

## 综合评分计算

**加权平均**（可根据项目类型调整权重）：

| 维度 | 默认权重 | 安全敏感项目 | 快速迭代项目 |
|------|---------|-------------|-------------|
| 依赖健康 | 15% | 20% | 10% |
| 文档完整 | 10% | 10% | 15% |
| 测试覆盖 | 20% | 15% | 25% |
| 安全态势 | 20% | 30% | 15% |
| 架构质量 | 20% | 15% | 15% |
| 代码质量 | 15% | 10% | 20% |

综合评分 = Σ(维度分 × 权重)
