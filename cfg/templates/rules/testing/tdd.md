# Testing & TDD

---

## Testing expectations (always apply)

* Every implementation must include a test plan or verification method.
* For bug fixes: write a failing test first when feasible; otherwise provide exact reproduction steps.
* For **complex / high-risk** changes: use strict TDD (Red-Green-Refactor) and require user review of tests before implementation.
* For the full playbook (integration-first, BDD fallback, deliverables), load the `testing-tdd` skill.
